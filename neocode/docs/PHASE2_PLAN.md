# Phase 2 实现计划

- 目标：在 Phase 1 的基础上，增强 REPL 的交互控制（plan/preview/apply），并引入更完善的本地化表达式处理、错误处理、以及简单的历史/回滚机制。
- 主要任务：
  1. REPL 入口扩展：新增对命令的显式解析（plan/preview/apply），并在状态机层面提供明确的错误分支。
  2. Edits 验证：对 Edits 的路径、操作和内容进行基本校验，避免不可落地的改动。
  3. 回滚支持：在落地前创建备份，修改失败时尝试回滚到备份版本。
  4. 历史与元数据占位：引入 History 结构，用于记录关键变更点，方便后续扩展。
  5. 测试覆盖扩展：增加 REPL 端到端测试的骨架，确保 plan/apply 的正确性。
- 验收标准：
  - 用户在 REPL 中可以通过 plan/preview 查看拟执行的改动，随后通过 apply 应用改动。
  - 对失败改动有回滚与清晰错误信息。
  - CI 能通过 go test 与 go build。
